Understanding def build_app() -> FastAPI:

The function that assembles the entire game show — from stage lights to the final answer.

Foundation Layer — Setting the Stage
$100
settings
The Game Manual — rules, stakes, clock
+
Analogy: Before any game show airs, producers consult the Game Manual — it defines the prize amounts, timing rules, and house rules. get_settings() loads all configuration (API keys, thresholds, DB paths) so every other component knows the rules of engagement.
main.py 
settings = get_settings()
$200
app
The Stage Itself — where the show is broadcast
+
Analogy: The Stage is the physical set — cameras, monitors, the hot seat. FastAPI() creates the web server that hosts every endpoint: the arena where contestants (users) interact with the show.
main.py 
app = FastAPI(title=settings.app_name, version="1.0.0")
$500
db
The Question Bank — stored trivia
+
Analogy: Every Millionaire episode needs a Question Bank — thousands of verified questions sorted by difficulty. DatabaseManager is the SQLite-backed store holding sessions, bookings, and API keys. Nothing can function without data to draw from.
main.py 
db = DatabaseManager(settings)
api_key_manager = APIKeyManager(db)
app.state.api_key_manager = api_key_manager
Services Layer — Knowledge & Output
$1K
pdf
The Official Check — generates the prize
+
Analogy: When a contestant wins, someone backstage prints the Official Check — the big cardboard one they hold for the cameras. PDFGenerator creates downloadable PDF documents (booking confirmations, itineraries) that are the tangible output of a successful interaction.
main.py 
pdf = PDFGenerator(settings.pdf_output_dir)
$4K
rag
The Deep Encyclopedia — obscure knowledge lookup
+
Analogy: For the hardest questions, producers consult a Deep Encyclopedia — multiple volumes of obscure facts, cross-referenced and indexed. AdvancedRAG combines vector search (ChromaDB), BM25 keyword matching, and a neural reranker to retrieve the most relevant resort knowledge for any question.
main.py 
rag = AdvancedRAG(
    settings.rag_persist_dir,
    settings.rag_collection,
    embedding_model=getattr(settings, "embedding_model", "all-MiniLM-L6-v2"),
    reranker_model="BAAI/bge-reranker-base",
    ingestion_token=settings.rag_ingestion_token,
)
Integration Layer — Lifelines & Memory
$16K
memory
Contestant's Progress — tracks used lifelines
+
Analogy: The production board tracks the Contestant's Progress — which lifelines are used, what safety nets remain, and which questions have been answered. MemoryStore maintains conversation history per session, so the chatbot remembers context across multiple turns.
main.py 
memory = MemoryStore(db=db)
$64K
registry
The Lifelines — Phone-a-Friend, Ask the Host
+
Analogy: The famous Lifelines — Phone-a-Friend, 50:50, Ask the Audience. Each is a callable action the contestant can invoke. make_registry creates the ToolRegistry with tools like search_rooms, create_booking, and rag_search. Each tool is a "lifeline" the agent can call for help.
main.py 
registry = make_registry(
    db=db, pdf=pdf,
    rag_search_fn=lambda query, k=5: rag.search(query=query, k=k)
)
Intelligence Layer — Decision & Verification
$125K
router
The Floor Manager — decides lock-in vs thinking
+
Analogy: The Floor Manager makes real-time calls — "Go to commercial," "Cue the music," "Is that your final answer?" The ModelRouter tries OpenAI first, falls back to Anthropic (Claude), then Ollama. It decides which LLM "locks in" the response, ensuring the show goes on even if one provider is down.
main.py 
# Build multi-model router
router = _build_router(settings)
$500K
detector
Final Answer Verifier — catches hallucinations
+
Analogy: Before the host declares "That's correct!", a Final Answer Verifier backstage double-checks every answer against the source material. HallucinationDetector scores the LLM's response against RAG context, flagging responses as HIGH, MEDIUM, or LOW confidence — ensuring no "wrong answers" make it to air.
main.py 
# Hallucination detector
detector = HallucinationDetector(
    high_threshold=settings.hallucination_high_threshold,
    medium_threshold=settings.hallucination_medium_threshold,
)
Orchestration Layer — Showtime
$750K
tracker
The Control Room — monitors ratings & tokens
+
Analogy: The Control Room monitors everything in real time — viewer ratings, ad breaks, camera feeds. MLflowTracker logs every interaction: latency, token counts, hallucination scores, and tool calls. If ratings drop (performance degrades), the producers know immediately.
main.py 
# MLflow tracker (graceful noop if disabled)
tracker = MLflowTracker(
    tracking_uri=settings.mlflow_tracking_uri,
    experiment_name=settings.mlflow_experiment_name,
    enabled=settings.mlflow_enabled,
)

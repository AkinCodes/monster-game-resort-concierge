request_id is popped from kwargs (or a fresh UUID is generated)

The Code
VALID_HOTELS = {"The Mummy Resort ...", ...}
A Python set defined at module level. Igor checks hotel_name not in VALID_HOTELS before touching the database.
Why it Matters
Without this check, a hallucinating LLM could book a guest into "Hotel Transylvania 4" or "Dracula's Secret Dungeon" — hotels that don't exist in the database. The registry is the bouncer at the door.

// How Counters differ from Variables
Counter Rules
A Prometheus Counter can only go up. It never decrements. It resets to zero only when the process restarts — which Prometheus tracks automatically via the rate() function.
Labels = Dimensions
.labels(tool=name) creates a separate time series per tool name. One counter in code, three series in Prometheus. You can graph each servant's call rate independently.

🌟 The Sesame Street Guide to tools.py 🌟
🎪
The Tool Dataclass
Tool(frozen=True)
The Concept

Each servant has a job card pinned to their workstation. It says their name, describes their job in one sentence, and has a function attached. The card is frozen — you cannot change it after setup.

"Grover has a job card: name = 'super-waiter', description = 'bring food to the table', fn = bring_food. He can't change his job mid-shift. That's frozen=True."
📋
JSON Schema
to_openai_schema()
The Concept

The LLM doesn't read Python. It reads JSON. to_openai_schema() translates each tool's requirements into a structured menu the Oracle can understand and follow.

"Oscar the Grouch only reads menus in very specific formats. You must write the specials in his exact format or he ignores you. The schema is the menu — written in Oscar's language."
🗂
ToolRegistry
The Operations Manager
The Concept

The registry is the clipboard that holds all the job cards. It has methods: list all tools, get a specific tool, execute with timing. The Oracle never calls Igor directly — it always goes through the registry.

"Gordon runs the neighbourhood. You don't run up to Elmo directly — you ask Gordon to find Elmo. Gordon knows where everyone is and makes sure the job gets logged."
🏗
make_registry()
The Factory Function
The Concept

You can't build a sandwich without ingredients. make_registry is the moment you hand the servants their tools — the database, the PDF printer, the RAG function. Without this call, the servants exist but are empty-handed.

"Mr. Hooper opens the shop, stocks the shelves, gives each worker their equipment. That's make_registry. No setup, no service."
⏱
async_execute_with_timing
The Stopwatch Wrapper
The Concept

Every servant task is wrapped in a stopwatch. Before the task: start clock, log request_id. After: stop clock, log elapsed_ms, log result. If something explodes: catch it gracefully and return a nice error dict.

"Telly Monster is very anxious and times everything. 'That took 342 milliseconds!' He writes it all down. Nothing happens without Telly knowing exactly when it started and stopped."
🔢
TOOL_CALL_COUNT
Prometheus Counter
The Concept

One counter, three time series. Labels let Prometheus track each tool separately while sharing the same metric name. The count never decreases — it's a cumulative total since the server started.

"The Count has three chalkboards on the wall — one per servant. 'Igor was called FORTY-TWO times! Ah ah ah!' The chalkboards only ever add. They never erase."
🔔
The Bell
The Oracle calling a tool
The Concept

The LLM doesn't execute code — it produces a JSON blob specifying which tool and which arguments. The concierge reads that blob and rings the appropriate servant's bell. The bell is async_execute_with_timing(name, **kwargs).

"Big Bird shouts the order ('I need a room booked!'). The Oracle translates that into structured JSON. The registry reads it and sends Igor running. Big Bird never touches the ledger directly."
🧹
clean_kwargs
Trailing colon stripper
The Concept

The LLM sometimes sends parameter names with trailing colons: "guest_name:". Before passing kwargs to the servant, the registry strips those colons so the function signature matches perfectly.

"Snuffleupagus sometimes mumbles with extra sounds at the end of words. The translator (clean_kwargs) quietly fixes it before passing the message on, so everyone understands."

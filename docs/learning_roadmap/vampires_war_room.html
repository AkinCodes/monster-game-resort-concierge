In The Switchboard, you learned how tools are registered like service desks and how the dispatcher routes calls to them. Each desk had its equipment wired in before opening. The dispatcher assigned tickets, started stopwatches, and logged everything.

Now: We zoom out. Before any call reaches the switchboard, someone has to decide whether it's even a switchboard call at all. That someone is the General. He reads the request first, picks a colored flag, and sends it to the right department. Planning and execution are separate jobs — two desks, one answer.

The Colored Flags

Every request that walks through the door gets classified. Big Bird keeps four colored flags in a rack on his desk. He reads the request, grabs the right flag, and attaches it to the paperwork. The flag tells everyone downstream what kind of job this is.

Big Bird — The General at His Desk

A guest walks in. Big Bird reads the request out loud, slowly. He looks at the flag rack on his desk — four flags, four colors, four meanings.

He grabs a flag:

The blue flag means "Go ask the library." The guest wants to know something — what time the pool opens, which restaurant has garlic-free options. This is a research job.

The red flag means "Go do something." The guest wants to act — book a room, look up a reservation. This goes straight to the service desk downstairs.

The yellow flag means "Go back and ask the guest for more details." The request is incomplete — they said "book a room" but didn't say which hotel or what dates. Can't route it anywhere yet.

The green flag means "Just chat." No departments needed. The guest said hello, or asked about the weather, or told a joke. Big Bird handles it right here at his desk.

The Planner

Agent 1: Big Bird reads the guest's request out loud to the voice upstairs. The voice sends back a form with the flag color and details. But sometimes the form comes back garbled.

Big Bird — Reading the Request Aloud

Big Bird picks up the guest's request and reads it out loud to the voice upstairs. The voice is wise — it listens to the request, considers what tools are available at the service desks downstairs, and sends back a form.

The form says: "This is a blue flag. The guest wants to know about pool hours. Search for 'pool hours' in the library."

But sometimes the form comes back garbled. The ink is smudged. The handwriting is illegible. Big Bird squints at it, holds it up to the light, and... can't read it.

So he falls back to his gut feeling. He reads the guest's original request again himself. Did the guest say "book"? That sounds like a red flag. Did they mention "spa" or "pool"? That's probably a blue flag. Can't tell at all? Green flag — just be friendly, you can't go wrong with a warm greeting.

🧠 The gut feeling is the fallback parser. When the voice upstairs sends back something unreadable, Big Bird doesn't panic — he scans the original request for keywords. "book" or "reserve" = red flag. "pool" or "spa" = blue flag. Everything else = green flag. It's crude, but it keeps the hotel running.

The Executor

Agent 2: The department that received the flag does the work. Four flags, four routes, four different jobs. Each one knows exactly what to do.

Big Bird — Dispatching the Flag

Big Bird holds up the flag he picked and calls out to the right department:

The blue flag goes to the Grand Library. The librarian searches the shelves, finds the relevant pages, and brings back an answer grounded in what the books actually say.

The red flag goes to the Service Desk downstairs. The desk clerk picks up the phone, executes the tool — books the room, looks up the reservation — and comes back with a result. Then Big Bird asks the voice upstairs to turn that raw result into a friendly sentence for the guest.

The yellow flag goes right back to the guest. "Could you tell me which hotel you'd like? And what dates work for you?" No departments needed — just a polite question to fill in the blanks.

The green flag stays right here. Big Bird chats warmly with the guest. No tools, no library, no service desk. Just conversation.

🔵 KNOWLEDGE
_execute_knowledge()
Run to the Grand Library, search the shelves, bring back an answer
▾
The Grand Library Route

The blue flag arrives at the Grand Library. The librarian takes the search query (or the guest's original question if no query was specified) and searches the shelves — up to 5 books at a time.

She stacks the relevant pages on the reading desk, notes which books they came from (the sources list), and calls the voice upstairs: "Here's what the books say. Answer the guest's question using ONLY this material. If it's not in the books, say so honestly."

The voice reads the pages, composes a response, and sends it back. The librarian wraps it all up — the response, the sources, and the cost — into a result package.

PYTHON async def _execute_knowledge(self, plan, user_message, session_id): # Search the library shelves query = plan.search_query or user_message rag_results = self.rag.search(query, k=5) # Stack the pages and note which books they came from docs = rag_results.get("results", []) context_parts, sources = [], [] for doc in docs: context_parts.append(doc["text"]) sources.append(doc["meta"]["source"]) # Ask the voice upstairs to answer using ONLY these pages prompt = EXECUTOR_KNOWLEDGE_PROMPT.format( context="\n---\n".join(context_parts), history=self._format_history(history), question=user_message, ) response = await self.llm.chat(messages) return ExecutionResult( response=response.content, plan=plan, sources=sources, )

The Service Desk Route

The red flag arrives at the Service Desk. But first — two safety checks.

Check 1: Did Big Bird actually write down which desk to go to? If the form says "do something" but doesn't specify what, the clerk politely says: "I understood you want to perform an action, but I couldn't determine which one."

Check 2: Does that desk even exist? If the form says "go to the Moon Desk" and there is no Moon Desk, the clerk lists what IS available.

If both checks pass, the clerk injects the guest's room number (session ID) into the request, fires off the job with the dispatcher's timing system, and gets a raw result back. Then the voice upstairs turns that raw result into a friendly sentence the guest can understand.

PYTHON async def _execute_tool(self, plan, user_message, session_id): # Safety check: did the form specify which desk? if not plan.tool_name: return ExecutionResult(response="Could you be more specific?", ...) # Safety check: does that desk exist? tool = self.tools.get(plan.tool_name) if not tool: return ExecutionResult(response=f"No tool '{plan.tool_name}'", ...) # Inject the guest's room number into the request args = dict(plan.tool_args) args["session_id"] = session_id # Fire off the job through the dispatcher tool_result = await self.tools.async_execute_with_timing( plan.tool_name, request_id=uuid4(), **args ) # Ask the voice upstairs to make it guest-friendly response = await self.llm.chat(summarize_prompt) return ExecutionResult(response=response.content, tool_result=tool_result, ...)

The Clarification Route

The yellow flag doesn't go to any department. It goes back to the guest.

Big Bird reads the planner's reasoning — "the guest wants to book but didn't specify which hotel or dates" — and asks the voice upstairs to compose a friendly, specific follow-up question. The voice checks the guest's recent conversation (in case the answer is already there) and writes something like: "I'd love to help you book! Could you tell me which hotel you'd like, and your check-in and check-out dates?"

PYTHON async def _execute_clarify(self, plan, user_message, session_id): history = self.memory.get_messages(session_id, limit=5) prompt = EXECUTOR_CLARIFY_PROMPT.format( reasoning=plan.reasoning, # what's missing history=self._format_history(history), question=user_message, ) response = await self.llm.chat(messages) return ExecutionResult(response=response.content, plan=plan)

The Chat Route

The green flag never leaves the War Room. No library, no service desk, no follow-up question. Big Bird just asks the voice upstairs to respond warmly and in character — as the Monster Resort Concierge.

The voice checks the guest's recent conversation to keep continuity, and writes a brief, helpful, friendly reply. That's it. The simplest route of all.

PYTHON async def _execute_chitchat(self, plan, user_message, session_id): history = self.memory.get_messages(session_id, limit=5) prompt = EXECUTOR_CHITCHAT_PROMPT.format( history=self._format_history(history), question=user_message, ) response = await self.llm.chat(messages) return ExecutionResult(response=response.content, plan=plan)
⚡ The execute() method is just a four-way switch. It reads the flag color on the plan and dispatches to the matching route. If anything goes wrong inside a route, the executor catches the error and sends back a polite apology instead of crashing. The guest never sees a stack trace — just "I'm terribly sorry, could you try rephrasing?"
PYTHON # Agent 2: Read the flag, send it to the right department async def execute(self, plan: Plan, user_message: str, session_id: str) -> ExecutionResult: start = time.monotonic() try: if plan.intent == IntentType.KNOWLEDGE: # blue flag result = await self._execute_knowledge(...) elif plan.intent == IntentType.TOOL: # red flag result = await self._execute_tool(...) elif plan.intent == IntentType.CLARIFY: # yellow flag result = await self._execute_clarify(...) else: # green flag result = await self._execute_chitchat(...) except Exception: # Something broke — apologize gracefully result = ExecutionResult( response="I'm terribly sorry, something went wrong...", plan=plan, ) result.latency_ms = (time.monotonic() - start) * 1000 return result

The Full Flow

The handle() method is the front door of the War Room. Every guest request enters here, and a complete answer walks out. Two desks, six steps, one answer.

1
Guest Walks into the War Room
A guest request arrives — a string of text and a room number. This is the raw input. Nothing has been decided yet. No flags, no departments, no plan.
handle(user_message, session_id)
2
Big Bird Reads the Request
Big Bird reads the request to the voice upstairs. The voice sends back a form with a flag color and details. If the form is garbled, Big Bird falls back to his gut feeling. Either way, a Plan comes out — a flag is picked.
plan = await self.plan(user_message, session_id)
3
Big Bird Picks a Flag and Sends It
The flag goes to the right department. Blue to the library, red to the service desk, yellow back to the guest, green stays here. The department does its work and returns a result.
result = await self.execute(plan, user_message, session_id)
4
The Night Watchman Checks (Library Answers Only)
If the flag was blue — a library answer — the Night Watchman inspects the response. Did the librarian make anything up? Is the answer grounded in what the books actually say? He scores the confidence level. For all other flag colors, this step is skipped.
if plan.intent == KNOWLEDGE: hal_detector.score_response(...)
5
Big Bird Writes in the Guest's File
The exchange is saved to memory: what the guest said, and what the concierge answered. Both go into the guest's file so the next conversation has context. The notepad grows by two lines.
memory.add_message(session_id, "user", ...) + memory.add_message(session_id, "assistant", ...)
6
Guest Receives the Answer
The ExecutionResult walks out the door. It carries the response text, which flag was picked, any tool results, library sources, timing data, and the Night Watchman's verdict. The guest sees only the response. Everything else is for the hotel's records.
return result
PYTHON # The front door of the War Room — every request enters here async def handle(self, user_message: str, session_id: str) -> ExecutionResult: overall_start = time.monotonic() # Step 1-2: Big Bird reads the request and picks a flag plan = await self.plan(user_message, session_id) # Step 3: Send the flag to the right department result = await self.execute(plan, user_message, session_id) # Step 4: Night Watchman checks library answers if plan.intent == IntentType.KNOWLEDGE: try: hal_detector = HallucinationDetector() result.confidence = hal_detector.score_response( result.response, result.sources, user_message ) except Exception: pass # Night Watchman is off duty — carry on # Step 5: Write the exchange in the guest's file self.memory.add_message(session_id, "user", user_message) self.memory.add_message(session_id, "assistant", result.response) # Step 6: Hand the answer to the guest return result

🏰 The War Room owns the full lifecycle of a request. It doesn't do the searching (the library does that). It doesn't do the booking (the service desk does that). It doesn't store the memory (the hourglass does that). It doesn't check for lies (the Night Watchman does that). But it coordinates all of them — in the right order, with the right flag, every single time.

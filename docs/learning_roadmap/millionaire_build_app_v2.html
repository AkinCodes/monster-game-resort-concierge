Understanding
def build_app() -> FastAPI:

A single function that assembles every moving part of the Monster Resort Concierge. We'll use Who Wants to Be a Millionaire? to decode each component — from the game manual to the million-dollar host.

build_app() is the stage crew that sets up the entire game show before a single question is asked — it wires the lights, loads the questions, hires the host, and opens the doors to the audience.

Game Show Role

The Game Manual

Before filming begins, the production team reads the game manual — it defines the rules, the stakes, the time limits, and every configurable detail of how the show runs. Without it, nobody knows the format.

In our app: settings loads all environment variables — API keys, thresholds, model names, database paths — via Pydantic's BaseSettings. Every component downstream reads from this single source of truth.

Game Show Role

The Deep Encyclopedia

Imagine a vast research library backstage that the host can consult for obscure facts — it doesn't just keyword-search; it understands meaning, ranks relevance, and surfaces the best passages.

In our app: AdvancedRAG combines ChromaDB vector search with BM25 keyword search, then applies a cross-encoder reranker to produce the most relevant resort knowledge for every guest query. Hybrid search, real intelligence.

Question 06 · $4,000

The Lifelines

Phone-a-Friend. Ask the Audience. 50:50.

📞

Game Show Role

The Lifelines

When a contestant is stuck, they reach for a lifeline — each one is a different strategy for getting help. The show pre-registers all available lifelines before the game starts.

In our app: make_registry() builds a ToolRegistry containing every tool the agent can invoke — booking rooms, searching the knowledge base, generating PDFs. Each tool has a name, a schema, and a callable. The LLM picks which lifeline to use.

Game Show Role

Final Answer Verifier

After the contestant says "final answer," there's a verification system that checks the answer against the master database. It catches mistakes before the host commits to "correct" or "incorrect."

In our app: HallucinationDetector compares the LLM's response against the RAG source material. It flags responses as HIGH, MEDIUM, or LOW confidence — catching hallucinations before they reach the guest.

Question 09 · $250,000

The Control Room

Monitoring ratings, timing, and everything behind the glass.

📺

Game Show Role

The Control Room

The control room sits behind soundproof glass, monitoring every metric: viewer ratings, segment timing, audio levels, camera feeds. Producers make real-time decisions based on this data.

In our app: MLflowTracker logs every interaction — latency, token usage, model choice, hallucination scores. It writes to MLflow for experiment tracking and gracefully becomes a no-op if disabled. Observability without overhead.

The Million Dollar Question

The Host

The one who delivers the final verdict.

🎤

Game Show Role

The Host — Regis Philbin

The host is the face of the show. They don't write the questions, manage the cameras, or verify answers — but they orchestrate the entire experience. They take input from every department and deliver one seamless performance to the contestant.

In our app: _agent_reply is the inner async function that ties everything together. It takes a user message, retrieves RAG context, builds the prompt, calls the LLM via the router, executes tool calls, runs hallucination detection, logs metrics, and returns the final response. It is the closure that captures every component assembled above — the host who knows the whole crew by name.
